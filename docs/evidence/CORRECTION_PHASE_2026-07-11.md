## 1) التغييرات المطبَّقة

### أ) مصدر حلقة نسبة التحصيل → `FIN.memberDelinquency`
كانت الحلقة تقرأ العمود المخزّن `balance_ils` لتحديد «مَن دفع». أصبحت تعتمد على
`FIN.memberDelinquency(memberId).byYear[cy].settled` (المبني على `due_amount_ils` مقابل
`paid_amount_ils`) — نفس تعريف السداد في المحرّك.

### ب) مقام مؤشّرات KPI (استبعاد الازدواج)
كان المقام `|fb|+|rd|+|np|+|db|`، و`np ≡ fb + rd` (صافي المركز = الرصيد + العجز)، فيُحتسب
`fb` و`rd` مرّتين. أصبح المقام `|fb|+|rd|+|db|` (الأرقام المستقلّة فقط).

## 3) مراجعة التتبّع — كل رقم معروض ← مصدره

| السطح | الرقم | المصدر (بعد التصحيح) |
|---|---|---|
| لوحة الخزينة | إجمالي الغداء/الديوان/التبرعات | `FIN.foodBalance / diwanBalance / donBalance` |
| مؤشّرات KPI | القيم الأربع | `FIN.foodBalance / foodDeficitRemaining / foodNetPosition / diwanBalance` |
| مؤشّرات KPI | نِسَب الأشرطة | مشتقّة من القيم أعلاه (مقام بلا ازدواج) |
| حلقة التحصيل | نسبة السداد | `FIN.memberDelinquency` |
| صفحة النسخ | الأعداد والأرصدة | `DB.*` (أعداد) + `FIN.foodBalance/diwanBalance` |
| كشف الصندوق | الحركات والإجماليات | `FIN.fundLedger` + `FIN.food*` |
| كشف العضو (شاشة/طباعة) | الحركات والرصيد | `FIN.memberStatement` (الرصيد المُرحّل = `historical_balance_ils − historical_payments_ils`، نفس مُدخلات المحرّك) |
| تقرير المديونية | الرصيد الحالي | `FIN.memberStatement.finalBalance` |
| تقرير المتأخرين | حالة التأخّر | `FIN.memberDelinquency` |
| كشف التبرعات | التوزيع | `FIN.allocateFoodDonations` + `FIN.food*Total` |
| جدول الأعضاء (طباعة) | الرصيد | `FIN.memberBalance` |

**النتيجة:** كل رقم مالي معروض يُتتبَّع الآن حتى دالة `FIN` واحدة، أو حتى عمود مصدر خام يستهلكه
المحرّك نفسه (الذمة/المدفوعات التاريخية) — لا مصدر ثانٍ متعارض.

## 5) ملاحظة للمراجعة (قرار تصميمي مفتوح)
بطاقة «صافي مركز الغداء» (`np`) هي **مركّب** (`fb + rd`)، فإظهار «% من إجمالي الخزينة» لها إلى جانب
مكوّنيها يبقى متداخلاً مفهومياً حتى بعد إصلاح المقام. التصحيح أزال الازدواج من المقام (أضيق إصلاح
ممكن). إن رغبت، يمكن لاحقاً حذف «% من الإجمالي» من بطاقة `np` تحديداً — وهو قرار تصميم لا تصحيح مصدر.
