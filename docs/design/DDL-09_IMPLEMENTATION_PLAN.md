# DDL‑09 — تطبيق لغة تصميم الديوان على النظام الفعليّ
# Diwan Design Language — Implementation Plan (design → code)

> **الحالة:** خطةٌ مقترحة — **لا تنفيذَ على كود الإنتاج قبل اعتماد المالك** (تغييرٌ بصريٌّ واسعٌ ظاهر).
> **النوع:** تحسين تجربة (تطبيق الهوية البصرية) ضمن Production Evolution. **واجهةٌ/إخراجٌ فقط.**
> **الحدّ الأحمر:** لا منطقَ أعمال، لا `FIN.*`، لا SQL، لا Supabase، **ولا تغيير رقمٍ ماليّ** — ضمانُ التطابق البايتيّ (غذاء 1694.04 · ديوان 3481 · عجز −8139 · صافي −6444.96). «لا يُعاد كتابة سندٍ تاريخيّ».

## 0) الوضع الحاليّ (فُحِص)
`public/css/app.css` (1309 سطراً) **مبنيٌّ على رموزٍ (CSS variables)** في كتل `:root` + `body.light` + الداكن. الحاليّ: علامة كُحليّة `#24425C`، أكسنت أزرق `#2F6DB3`، أكسنت كشف الحساب فيروزيّ `#0E9E84`، موجب/سالب أخضر/أحمر، وألوان تصنيفٍ للخزائن. **⇒ جوهر التطبيق هو إعادة ربط هذه الرموز بقيم DDL‑02 المُجمّدة**، ثم صقلُ الشكل (DDL‑03) صفحةً صفحة — وأغلبه محتوىً في تعريفات الرموز، قابلٌ للعكس.

## 2) الضمانات (كل مرحلة)
- **تطابقٌ بايتيٌّ للأرقام المالية** قبل/بعد (نفس منهج Foundation‑B).
- **قبول 48/48** (`node tests/e2e-acceptance.cjs 3273`) بعد كل مرحلة.
- **لا تغييرَ منطقٍ**؛ فقط CSS ومَلمَحُ markup حيث يلزم (وسم القطبية، إزالة حاويةٍ زائدة).
- **الوضعان** (فاتح/داكن) و**RTL** و`reduced-motion` محفوظة.
- **قابليةُ العكس** لكل مرحلةٍ على حِدة (revert مركّز).
- **مراجعة Fable 5** المستقلّة عند المرحلتين الحسّاستين (الألوان الحيّة، والطباعة).

## 3) التحقّق
`node --check` · مجموعة القبول (منفذ 3273) · لقطاتٌ بصريةٌ فاتح/داكن للصفحة · مقارنةُ الأرقام قبل/بعد.

## 4) قراراتٌ مطلوبةٌ قبل التنفيذ
1. **اعتماد البدء بالمرحلة 1** (إعادة ربط الرموز) على كود الإنتاج — الأعلى أثراً والأقلّ خطراً.
2. **تأكيد ترتيب الطرح** (صفحات العمل أعلاه) — أو تعديله.
3. **بنية العمل:** مرحلةٌ واحدةٌ لكل PR (أوضح للمراجعة والعكس) أم تراكمٌ على فرعٍ واحد؟

**عند اعتمادك:** أبدأ المرحلة 1 (إعادة ربط الرموز)، أُشغّل القبول والتطابق البايتيّ، وأعرض النتيجة بلقطاتٍ قبل التوسّع.
