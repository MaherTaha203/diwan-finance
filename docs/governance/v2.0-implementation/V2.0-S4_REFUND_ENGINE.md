# V2.0 · Slice 4 — Refund Engine (CA‑005 · BEHIND FLAG · default OFF)

**Phase:** MODEL2 V2.0 Implementation · **Slice:** 4 · **Date:** 2026‑07‑23 · **Baseline:** `main` @ `16269f7`
**Constitutional basis:** CA‑005 · ADR‑005 · OD‑05.

## 1 · Objective
Implement the frozen **refund** policy as a **first‑class movement** — a NEW voucher reflecting money
leaving, linked to its origin receipt — **not** a cancellation. Full or partial, currency‑preserving,
funded always from the **origin treasury**, prohibited after period close. Delivered pure + flag‑guarded so
the system stays **byte‑identical to today** until Slice 7 activation.

## 3 · Behaviour introduced (only when flag ON — Slice 7)
`BusinessOps.refundReceipt(...)` creates a NEW refund payment (outflow, `movement_type:'refund'`) linked to
the origin receipt, drawn from the origin treasury, full or partial, currency‑preserving. The origin receipt
is untouched (Law 5); value is conserved (Law 1); the refund is fully audited (Law 6). `fin2.treasuryBalance`
already nets the refund outflow against the origin treasury (the `refund` event is `outflow:true`), so a full
refund returns the origin treasury to its pre‑receipt position.

**Cancellation vs Refund (now distinct):** BO‑03 cancellation remains the *pre‑finalization preserved void*
(FOC‑019); BO‑11 refund is the *first‑class post‑receipt outflow* (FOC‑013).

## 4 · Eligibility / "irreversible consequences" mapping
Per ADR‑005 the "final‑effects" boundary maps to the eligibility rule (a BO‑design detail). Slice 4 realises
it as: **origin is an active receipt, its period is not locked, and the requested amount ≤ the remaining
refundable (origin ILS − already‑refunded).** This is a conservative engineering realisation, not a
constitutional invention; a richer irreversibility trigger can be layered later with no code change while the
flag is OFF.

## 5 · Migration impact
None. Forward‑only, additive. No schema change; no data transform. With the flag OFF every refund attempt
returns `E_DISABLED` and no live UI path calls the operation, so the DB is untouched and behaviour is
byte‑identical to today.
