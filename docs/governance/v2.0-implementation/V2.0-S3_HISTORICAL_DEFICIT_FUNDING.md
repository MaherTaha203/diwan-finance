# V2.0 · Slice 3 — Historical Deficit **Funding** (CA‑004 R1)

**Phase:** MODEL2 V2.0 Implementation · **Slice:** 3 · **Date:** 2026‑07‑23 · **Baseline:** `main` @ `d2fc04a`
**Constitutional basis:** CA‑004 **Revision 1** (owner‑ratified) · resolves the Slice‑3 STOP.

## 1 · Objective
Align the implementation with **CA‑004 R1**: retire the forbidden internal **Historical Deficit Settlement**
outflow, and reflect the **funding‑only** model — the Historical Deficit Treasury is **informational** and
accumulates **Historical Deficit Funding** produced automatically by the allocation/collection paths. No
settlement transaction, treasury outflow, voucher, or accounting movement is recorded for external creditor
payments (they happen outside the software).

## 3 · Behaviour introduced
**None on production data.** No settlement row has ever existed (there was no write path), so every settlement
aggregate was already `0`; removing the concept is numerically inert. The change is a **model correction +
report relabel**: the deficit view now reads Opening Deficit · Treasury Balance · **Total Historical Funding**,
and the model no longer defines an outflow that the approved workflow forbids. Funding accumulation itself
(ق4 collections, ق5 debt‑settled transfers, directed donations → deficit treasury) is unchanged.

## 4 · Migration impact
None. No schema change; no data transform; forward‑only. The `MODEL2_ALLOCATION_ENABLED` flag is untouched and
irrelevant here (deficit funding already operates unflagged via the certified collection paths).
