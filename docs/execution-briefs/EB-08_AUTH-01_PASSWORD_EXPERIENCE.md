# Execution Brief — EB-08 · AUTH-01 — تجربة تغيير كلمة المرور + نظام تصميم المصادقة
# Production Evolution · New Feature — Professional password experience + Authentication Design System

> **الحالة:** مواصفةٌ مملوكةٌ للمالك (وثيقة TASK المرفوعة 2026-07-14) — نُفِّذت بتوجيهه المسبق «إذا طلبت ميزةً جديدة نفّذها»،
> وتخضع للدورة: تنفيذ → تحقّق → مراجعة مستقلّة → Closure Report → **اعتماد المالك النهائي قبل الدمج**.
> **النوع:** New Feature (مصادقة/واجهة) — **لا مساس بأي منطقٍ ماليّ** (FIN/FIN2/FinContract/MODEL2 خارج النطاق كلياً).

## 1) Business Scope
النظام يحمي بياناتٍ ماليةً عائلية؛ كلمات المرور هي خطّ الدفاع الأول. المطلوب تجربة تغييرٍ عالمية المستوى
(بمستوى Stripe/Linear) **بنفس لغة تصميم الديوان**، مع **نظام تصميم مصادقةٍ موحّد** يغطي: تغيير كلمة المرور،
الدخول الأول الإلزاميّ بكلمةٍ مؤقتة، إعادة التعيين عبر البريد، الدعوة، التحقق من البريد، وإنشاء كلمة المرور لأول مرة.

## 2) ما بُني (المكوّنات)
| المكوّن | الملف | الوصف |

| **AuthDS** — محرّك السياسة والواجهات | `public/js/auth-password.js` (جديد) | `checkPassword` (8 قواعد حيّة) · مؤشّر قوّة 5 مستويات · بصمات آخر 5 كلمات (SHA-256 مُملَّحة عبر Web Crypto في user_metadata) · مولّد كلمات قوية · `attachPolicyUI` (قائمة حيّة + قوّة + تطابق + بوّابة حفظ) · تجربة الشاشة الكاملة |

| القديم | `auth.js` / `index.html` | حُذف مودال 6-أحرف القديم؛ `window.changePassword` صار يفتح التجربة الجديدة (توافقٌ خلفيّ) |

## 3) سياسة كلمة المرور (مُطبَّقة حيّاً وعند الحفظ)
≥12 حرفاً · حرف كبير · حرف صغير · رقم · رمز خاص · لا تحوي اسم المستخدم · لا تحوي البريد (المقاطع ≥3 أحرف) ·
ليست كلمةً محظورة (password/admin/123456/qwerty/welcome/diwan/taha/…) ولا من قائمة ~60 كلمةً شائعة ·
**منع إعادة آخر 5 كلمات** (بصماتٌ مُملَّحة لكل مستخدم — «إن كان مدعوماً» تحقّق) · التجزئة الآمنة في خادم Supabase Auth.

## 4) الأمان
التحقق من كلمة المرور الحالية عبر **عميلٍ معزول** (لا يمسّ جلسة المستخدم) · تسجيل كل تغييرٍ في **Audit Log** ·
تصفير علم `must_change_password` ذرّياً مع التغيير · احترام معمارية المصادقة القائمة (Supabase Auth, RLS, أدوار).

## 5) إتاحة واستجابة
أهداف لمس ≥40px · تنقّل لوحة مفاتيح (Enter يحفظ، Tab محاصر، ESC محكوم بالوضع) · `aria-live` للقائمة والقوّة والتطابق ·
`role=dialog/alert` · تباينٌ عالٍ بالوضعين الفاتح والداكن · حالات تركيزٍ مرئية · `prefers-reduced-motion` · هاتف = بطاقةٌ بملء الشاشة بلا تمرير أفقيّ.

## 6) التحقّق (الدليل)
- `node --check` نظيف (auth-password/auth/app/reports/print).
- **قبول E2E: 48/48 · pageerrors=0** (لا انحدار — لا مساس بالمنطق الماليّ).
- **اختبار وظيفيّ مخصّص: 27/27 PASS** (السياسة ×9 · الشاشة والبوّابة ×10 · القفل ×3 · الدعوة ×2 · الاستعادة · الهاتف) + صفحة الاستعادة 8/8 قواعد وبوّابة حفظ.
- لقطات: فاتح/داكن · عاديّ/مقفل · دعوة · استعادة · هاتف 390px.

## 7) قيود معروفة
- منع إعادة الاستخدام ببصماتٍ في `user_metadata` (الطبقة المتاحة دون خادمٍ وسيط) — بصمات مُملَّحة لا كلمات؛ التجزئة الفعلية للاعتماد تبقى في خادم Supabase.
- أيقونات Tabler وخط IBM Plex عبر CDN (سلوك النظام القائم).
