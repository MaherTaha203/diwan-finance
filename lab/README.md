# المختبر الدستوري — Constitutional Laboratory

بيئةٌ **دائمة معزولة عن الإنتاج** للتحقّق من كل عملية مالية قبل أي تغيير وبعده. تُشغّل **التطبيق الحقيقي** فوق **مخزَنٍ في الذاكرة ببيانات بذرةٍ ثابتة**، وتحجب **كل اتصالٍ خارجيّ** — فلا تلمس قاعدة الإنتاج ولا بياناتها إطلاقًا.

## لماذا (الأغراض السبعة)
مركز توثيق · مركز تحقّق ماليّ · مجموعة اختبارات قبول · مجموعة اختبارات انحدار (Regression) · مركز تحقّق أعمال · مركز امتثال دستوري · **أرشيف أدلّة بلقطات شاشة** — بيئةٌ واحدة تخدمها كلّها.

## البنية
```
lab/
├── README.md                     ← هذا الملف (البنية + القواعد + كيفية التشغيل)
├── seed/
│   └── constitution-seed.json    ← بذرة ثابتة: 4 أعضاء اختبار بأوضاع مُعرَّفة (لا بيانات إنتاج)
├── cases.cjs                     ← الحالات الدائمة: كل حالة تربط فصل FOC بسيناريو + النتيجة المتوقَّعة
├── static.cjs                    ← خادم ملفّاتٍ محليّ لـ public/ (محليّ فقط)
├── run.cjs                       ← المُشغِّل: تصفير → لقطة قبل → تنفيذ بالواجهة الحقيقية → لقطة بعد → مقارنة متوقَّع/فعلي → تقرير
├── screenshots/<FOC-ID>/         ← أرشيف الأدلّة (before-*.png / after-*.png)
└── results/                      ← مخرجات التشغيل: REPORT.md + <FOC-ID>.json (متوقَّع↔فعلي)
```

## العزل عن الإنتاج (ضمانة صلبة)
- المخزَن كلّه **في الذاكرة** (بديلٌ لـ Supabase)، يُهيَّأ من `seed/constitution-seed.json` عند كل حالة.
- تسجيل الدخول والطباعة والشبكة **مُزيَّفة**؛ ومُشغِّل المتصفّح **يحجب كل طلبٍ ليس `localhost`** (`context.route → abort`)، فيستحيل الوصول للإنتاج.
- لا يُستعمل مفتاح إنتاجٍ ولا بياناته. كل تشغيلٍ يبدأ من نفس البذرة → **نتائج قابلة للتكرار**.

## كيف تُشغِّل المختبر (في أي وقت)
```bash
node lab/run.cjs                 # كل الحالات
node lab/run.cjs FOC-001 FOC-003 # حالات مُنتقاة
```
- المخرجات: `lab/results/REPORT.md` (ملخّص الأدلّة) + `lab/results/<FOC-ID>.json` (قبل/بعد/التحقّقات) + لقطات في `lab/screenshots/<FOC-ID>/`.
- **رمز الخروج (exit code):** صفر إذا طابق الفعليُّ المتوقَّع في كل الحالات، وغير صفر إذا تغيّرت أي نتيجة متوقَّعة (**بوّابة الانحدار**).

## حماية الانحدار (Regression)
قبل دمج أي تغيير في الكود المالي، شغّل `node lab/run.cjs`. إن تغيّرت أي نتيجة متوقَّعة، يفشل التشغيل ويوضّح التقرير أي **فصل FOC** تأثّر وما القيمة التي تبدّلت (متوقَّع↔فعلي) — لا يُصلَح تلقائيًّا.

## اكتشاف خلل (Bug) لا يُصلَح تلقائيًّا
إذا خالف السلوكُ الفعليّ المتوقَّعَ الدستوري، **لا يُصلِح المختبر شيئًا**؛ يُظهر: المتوقَّع، الفعلي، الأدلّة (JSON + لقطات)، والفصل المتأثّر — ليقرّر المالك.

## إضافة فصل FOC جديد (حالة دائمة)
أضِف كائنًا إلى `cases.cjs`: `{ id, title, member, narrative, op, expect(before, after) }`.
- `op`: العملية عبر الواجهة الحقيقية، مثل `{ type:'receipt', fund:'food', payerType:'member', member:'LAB-001', amount:400 }` أو تبرّع `{ fund:'donation', kind:'cash', display:'food', alloc:'support_current'|'reduce_deficit' }`.
- `expect`: دالّةٌ تُعيد قائمة تحقّقات `{ label, pass, detail }` مقارنةً بلقطتَي before/after (الخزائن، أرصدة الأعضاء، السجلّات، السند الأخير).

## الحالات المُغطّاة حاليًّا (خضراء)
`FOC-001` سداد اشتراك تامّ · `FOC-003` دفعة زائدة مع ذمّة · `FOC-004` تبرّع غداء صافٍ · `FOC-005` تبرّع عضوٍ مَدين (أولوية الدَّين ق5) · `FOC-006` تبرّع موجَّه للعجز. البنية عامّة لإضافة بقية فصول FOC.

## قواعد المختبر
- لا تنفيذ على الإنتاج، ولا تلويث بياناته — بحكم البناء (حجب الشبكة + مخزَن ذاكرة).
- تصفيرٌ قبل كل حالة؛ كل تشغيلٍ من نفس البذرة؛ كل نتيجةٍ قابلة للتكرار.
- المختبر **لا يعدّل** أي كودٍ ماليّ أو أثرٍ مجمَّد، ولا يُفعّل MODEL2.

*بنيةٌ هندسية إضافية للتحقّق فقط — معزولة تمامًا عن الإنتاج.*
