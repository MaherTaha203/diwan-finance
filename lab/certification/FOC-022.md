# FOC-022 — تحويل فائض العجز للغداء + دفعةٌ زائدة تُنشئ رصيدًا دائنًا (Overflow + Credit)

> **الحالة: CERTIFIED** · تاريخ الشهادة: 2026-07-22 · مُولَّد من تشغيلٍ حقيقيّ في المختبر الدستوري.

## القصّة
محمد أحمد (عليه 750) يُقبَض منه تحصيلُ ذمّةٍ موجَّهٌ للعجز بمبلغ 9000 (العجز المتبقّي 3000). يُسدَّد رصيدُه المستحقّ بالكامل أوّلًا (750 → 0)، ثمّ يُسجَّل الباقي 8250 رصيدًا دائنًا للعضو؛ ويُسدَّد العجز المشترك (0) ويتحوّل فائضه 6000 للغداء.

## الشرح التجاري (بالعربية المبسّطة)
أمران دستوريّان في عمليةٍ واحدة. (أ) الفائض المشترك: المبلغ الموجَّه للعجز أكبر من المتبقّي، فأُغلق العجز المشترك كاملًا (−3000 → 0) وتحوّل الفائض 6000 تلقائيًّا للغداء بقاعدة قراءةٍ لا صفوف تُكتب (ق2 الاشتقاق، ق9 حدّ العجز). (ب) الدفعة الزائدة للعضو (قرار المالك — توضيح دستوريّ): المقبوض 9000 أكبر من المستحقّ 750، فأُغلق المستحقّ بالكامل أوّلًا (صار 0) ثمّ سُجِّل الباقي = المقبوض − المستحقّ = 8250 رصيدًا دائنًا للعضو (ذمّةً سالبة). الرصيد الدائن لا يضيع ولا يُعامَل خطأً ولا يبقى معلّقًا بدَينٍ مُغلَق — يبقى ملكًا للعضو ويشارك في التسوية عند توليد الاشتراكات القادمة. الديوان لم يتغيّر.

| القيمة | قبل | بعد |
|---|---|---|
| خزينة الغداء | 0 | 6000 |
| خزينة الديوان | 5000 | 5000 |
| العجز المتبقّي | -3000 | 0 |
| رصيد LAB-001 | 750 | -8250 |
| سجلّ التبرّعات (عدد) | 0 | 0 |
| سجلّ القيود (audit) | 0 | 0 |

## المتوقَّع ↔ الفعلي (كل تحقّق)

- ✅ العجز المشترك −3000 → 0 (سُدّد بالكامل) — `-3000 → 0`
- ✅ الفائض المتحوّل للغداء = 6000 — `overflow=6000`
- ✅ خزينة الغداء 0 → 6000 (استلمت الفائض) — `0 → 6000`
- ✅ المستحقّ على العضو أُغلق بالكامل أوّلًا (750 → 0 مستحقّ) — `750 → مستحقّ 0`
- ✅ رصيدٌ دائنٌ للعضو = المقبوض − المستحقّ (9000 − 750 = 8250) — `credit=8250`
- ✅ الرصيد الدائن محفوظ (لم يضِع ولم يُعامَل خطأً) — `finalBalance=-8250`
- ✅ الديوان لم يتغيّر — `5000`
